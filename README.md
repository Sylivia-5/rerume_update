# rerume_update
一、 專案背景與動機
問題意識：因應18週學制導致畢業生較晚進入求職市場，許多理想職缺已被其他更早畢業的學生取得。

核心需求：新鮮人缺乏實戰經驗，難以將自身能力轉化為具說服力的履歷內容，需要有效方法凸顯個人特色與綜效。

二、 專案目的
主要目標：利用生成式AI模型，將使用者輸入的學經歷，自動美化、優化為數據化、專業且凸顯個人特色的履歷自傳。

預期成果：幫助求職者，特別是社會新鮮人，製作出能吸引企業目光的亮眼自傳。

三、 技術實作與流程
開發環境：使用 Google Colab 雲端平台進行開發。

關鍵技術：

後端：透過 Groq API 高速呼叫多種大型語言模型。

前端：使用 Gradio 函式庫快速建立簡易的網頁輸入與輸出介面。

運作流程：

使用者輸入「應徵職位」與「個人經歷」。

程式將輸入資訊與預先設定的「專業履歷撰寫助手」系統提示結合。

呼叫所選的語言模型生成自傳內容。

將美化後的自傳回傳至網頁介面顯示。

四、 模型比較與評估
測試模型：

模型1: deepseek-r1-distill-llama-70b

模型2: qwen/qwen3-32b (阿里巴巴中文特化模型)

模型3: llama3-8b-8192

模型4: meta-llama/llama-4-maverick-17b-128e-instruct

評估面向（根據人資觀點）：

專業術語與量化指標：模型2表現極佳，能提供具體數字（如時程偏差率5%）；模型1模糊；模型3、4空洞或失敗。

方法論與邏輯架構：模型2能展現系統化思考與執行細節；其餘模型方法論薄弱或無說明。

敘事流暢與個人特質：模型2敘事自然有層次；模型1平淡；模型3、4格式破碎或乏味。

對職位的理解深度：模型2能深入理解職位核心挑戰；其餘模型僅觸及表面。

綜合勝出模型：qwen/qwen3-32b，在所有評估面向上均表現最優，產出的自傳邏輯清晰、量化具體、個人形象鮮明。

五、 核心發現與結論
語言關鍵性：以中文為主要訓練語料的模型（如Qwen），在理解中文語境、職場文化與生成流暢自傳上，顯著優於以英文為主的模型。

模型演進：LLaMA-4對中文的處理能力已較LLaMA-3進步，但若追求高品質，仍應優先選擇中文特化模型。

實務建議：

對求職者：在中文職場求職，務必選用中文訓練為主的模型，以避免產出語意不順或內容空泛的自傳。

對人資/面試官：中文語境的準確度是判斷求職者真實實力的關鍵之一，可藉此初步篩選AI生成的罐頭履歷。
